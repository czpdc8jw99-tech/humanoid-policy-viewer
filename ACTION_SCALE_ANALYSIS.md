# Action Scale 分析

## 当前配置
- `action_scale`: 0.25
- `action_squash`: "tanh"
- `action_clip`: 100.0

## 问题分析

用户反馈：机器人可以稍微挣扎一会，然后往前倒下，机器人似乎没有任何动作。

可能的原因：
1. **tanh 把动作压得太小**：tanh 输出范围是 [-1, 1]，再乘以 0.25，最终动作范围只有 [-0.25, 0.25]
2. **策略在 command=0 时输出本身就很小**：训练时可能期望 command=0 时保持站立，但需要微小的调整动作
3. **初始姿态可能不完全匹配训练时的初始状态**：即使设置了 default_joint_pos，可能还需要一些时间让策略适应

## 解决方案

### 方案 A：增加 action_scale（最简单）
将 `action_scale` 从 0.25 增加到 0.5 或 1.0，让动作幅度更大。

### 方案 B：检查策略输出的原始值
如果策略输出的原始值（tanh 之前）就很小，那可能需要检查策略本身。

### 方案 C：调整 tanh 的应用时机
也许不应该对所有策略都应用 tanh，或者 tanh 应该在 action_scale 之后应用。

## 建议

先尝试方案 A：将 `action_scale` 增加到 0.5，看看是否能改善。
